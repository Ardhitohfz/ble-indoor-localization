# Data Directory

This directory follows the **Cookiecutter Data Science** project structure standard.

## Structure

```
data/
├── raw/                    # Original, immutable data (25 CSV files: A1-E5)
│   ├── A1.csv
│   ├── A2.csv
│   ├── ...
│   └── E5.csv
├── archive/                # Archived datasets (not used by default pipeline run)
│   └── raspi4_sink_legacy/ # Historical files moved from raspi4_sink/data/dataset
├── processed/              # Preprocessed data ready for modeling
│   ├── train.csv          # 80% training set (1,000 samples)
│   ├── test.csv           # 20% test set (250 samples)
│   └── preparation_summary.txt
├── interim/                # (Future) Intermediate transformations
└── rssi_boundaries.json   # Configuration for boundary constraints
```

## Directory Purposes

### `raw/`

**Canonical source data** - The primary dataset used by the ML pipeline.

- **Content**: 25 CSV files (5 cells × 5 repetitions: A1-E5)
- **Format**: RSSI readings from 4 BLE beacons + Cell label
- **Rule**: simpan data mentah sebagai sumber utama (hindari duplikasi lintas modul)
- **Collector default sink**: `raspi4_sink/rpi_sink_parallel.py` menulis CSV baru ke folder ini
- **Size**: ~1,250 samples (50 samples per cell)

### `processed/`

**Preprocessed, analysis-ready data** - Output of preprocessing pipeline.

- **Generated by**: `training/prepare_dataset.py`
- **Content**:
  - `train.csv`: 80% split for model training (1,000 samples)
  - `test.csv`: 20% split for final evaluation (250 samples)
  - `preparation_summary.txt`: Split statistics
- **Gitignored**: ✅ These files are regenerated on each run
- **Use**: Direct input to model training and evaluation

### `interim/` (Future)

**Intermediate transformations** - For complex preprocessing pipelines.

- **Purpose**: Store partially processed data between stages
- **Example use cases**:
  - Feature engineering experiments
  - Data cleaning checkpoints
  - Transformation comparisons
- **Status**: Not currently used (may be added later)

### `archive/`

**Historical datasets** - preserved for traceability, excluded from default pipeline input.

- **Content**: legacy sink files that differ from canonical `raw/`
- **Use**: manual comparison, audit, or migration
- **Default pipeline behavior**: reads from `raw/` unless `--data-dir` diubah

## Version Control

**.gitignore settings**:

- ✅ **Tracked**: `raw/` (single source of truth dataset)
- ❌ **Ignored**: `processed/` (regenerated on each run)
- ❌ **Ignored**: `interim/` (temporary transformations)

**Rationale**:

- Raw data is small (~125 KB) and essential for reproducibility
- Processed data is large and easily regenerated from raw data
- Commit raw data ensures anyone can reproduce the pipeline from scratch

## Data Lineage

|------|--------|---------|---------|
| `A1.csv` → `E5.csv` | Manual collection | Raw BLE RSSI readings | 50 each |

| `test.csv` | 20% of raw data | Final evaluation | 250 |

## Best Practices

1. **Never modify `raw/` files** - Keep original data pristine
2. **Regenerate `processed/` when raw data changes**
3. **Document data collection** in project README
4. **Track data versions** if collecting new samples
5. **Validate data integrity** before preprocessing
